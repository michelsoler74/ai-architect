# Política de Seguridad - AI Architect

En **AI Architect** nos tomamos en serio la integridad de nuestros datos de simulación y de los perfiles en Supabase. Si encuentras una vulnerabilidad en este repositorio o tienes dudas sobre un posible vector de ataque (como que las API Keys de OpenRouter/Groq pudieran quedar expuestas en el código cliente), háznoslo saber inmediatamente.

## Versiones Soportadas

| Versión | ¿Está Soportada? | Notas              |
| ------- | ---------------- | ------------------ |
| > v0.1.X| ✅ Sí            | Fase Beta Activa   |
| < v0.1.0| ❌ No            | Versiones Antiguas |

## Cómo Reportar una Vulnerabilidad

**Por favor, NO reportes vulnerabilidades creando un "Issue" público en GitHub.**

Preferiríamos que nos envíes un correo a la dirección de mantenimiento del equipo (por ejemplo, a través del perfil del arquitecto que encabeza el proyecto). 

Si consideras que la vulnerabilidad expone gravemente las variables de entorno de Supabase o credenciales LLM, indícalo claramente con la etiqueta "URGENTE". Proporcionaremos feedback sobre el problema en unas 48-72 horas y publicaremos las recompensas (o agradecimiento público) si se requiere un arreglo serio.
